ESREBORN WORLD CONTENT BLUEPRINT

의사가 배우는 AI 콘텐츠는
폴더가 아니라 메타데이터로 관리하는 편이 맞습니다.

세부 주제, 전문과, 난이도, 포맷, 키워드, 해시태그가 계속 늘어나는 구조에서는 폴더 트리를 깊게 만드는 순간 운영이 무너집니다. S-Reborn용 콘텐츠는 상단 frontmatter로 문서를 식별하고, 폴더는 단순 보관 단위로 유지하는 방식이 가장 실용적입니다.

권장 폴더 전략

s-reborn-world-doctor-ai/ 아래에 HTML 원고를 모으고, 분류는 파일 내부 frontmatter에서 처리합니다.

운영 기준

주제별 폴더 분기 대신 topic_groups, keywords, hashtags, audience를 배열로 관리합니다.

확장 방식

동일 글이 여러 카테고리에 걸쳐도 중복 저장 없이 메타데이터 검색만으로 재사용할 수 있습니다.

1. 추천 운영 구조

폴더 s-reborn-world-doctor-ai/ 단일 폴더
파일명 규칙 번호-콘텐츠성격-짧은주제.html
예: 11-doctors-learning-ai-intro.html
실제 분류 방식 frontmatter 기준 필터링
폴더 안 파일 종류 기획서, 템플릿, 실제 원고, 시리즈 페이지
핵심 원칙: 파일은 한곳에 두고, 의미는 메타데이터로 부여합니다.

2. Frontmatter 설계 기준

아래 필드는 최소 세트로 두면 운영이 편합니다.

필드 역할 예시
content_id 중복 없는 문서 ID doctor-ai-001
title 문서 제목 의사가 AI를 배워야 하는 이유
slug URL/시스템 식별용 why-doctors-should-learn-ai
content_type 글 종류 article, planning, guide, series
audience 주 독자 ["resident", "specialist"]
difficulty 난이도 beginner, intermediate
topic_groups 대주제 ["ai literacy", "clinical workflow"]
keywords 검색 키워드 ["의사 AI", "프롬프트", "차트 요약"]
hashtags 채널별 해시태그 ["#의사가배우는AI", "#의료AI"]
status 작업 상태 draft, review, published

3. 콘텐츠 기획 방식

메인 시리즈 구조

  • AI를 전혀 모르는 의사도 읽을 수 있게 입문 → 실무 → 윤리 → 고급 활용 순서로 구성합니다.
  • 한 편이 너무 넓어지지 않도록, 글마다 학습 목표를 한 문장으로 고정합니다.
  • 의학 지식이 아니라 업무 장면 중심으로 풀어야 반응이 좋습니다. 예: 차트 정리, 설명문 작성, 회의 준비, 환자 교육 자료 생성.

권장 콘텐츠 축

AI 리터러시

AI의 개념, 한계, 환각, 데이터 보안, 의사의 판단과 AI의 역할 구분

진료 워크플로우

문서 초안, 요약, 설명 자료, 커뮤니케이션 문안, 학회 발표 정리

교육과 연구

전공의 교육, 문헌 요약, 연구 아이디어 구조화, 강의자료 뼈대 생성

윤리와 검증

개인정보, 임상 판단 책임, 편향, 결과 검증, 사용 금지 영역

4. 세부 주제 제안

시리즈 세부 주제 핵심 키워드
입문 의사가 AI를 배워야 하는 이유 의료 AI, 생성형 AI, 업무 생산성, 디지털 전환
입문 AI가 잘하는 일과 못하는 일 환각, 근거 검증, 패턴 인식, 판단 책임
실무 외래 전후에 AI를 쓰는 방법 문진 정리, 설명문 초안, 환자 커뮤니케이션
실무 차트와 회의 내용을 요약하는 프롬프트 SOAP, EMR, 요약, 문서 자동화
교육 전공의와 교수의 AI 학습법 차이 전공의 교육, 피드백, 학습 설계
연구 논문 읽기 전에 AI로 구조를 파악하는 법 문헌 탐색, 연구 질문, 논문 요약
윤리 의료 현장에서 AI를 사용할 때 지켜야 할 선 개인정보, 책임소재, 보안, 검증

5. 키워드와 해시태그 운영법

키워드 설계 원칙

  • 한 문서에 메인 키워드 3개, 보조 키워드 5개 내외로 제한합니다.
  • 전문과 키워드와 행위 키워드를 함께 둡니다. 예: 내과, 설명문 작성, 차트 요약.
  • 유사어는 keywords에 같이 저장하고, 대표 분류는 topic_groups에서 통일합니다.

6. 문서 작성 규칙

  • 도입부에서 "왜 의사에게 필요한가"를 먼저 제시합니다.
  • 본문은 개념 설명보다 실제 사용 장면으로 전개합니다.
  • 각 글 마지막에는 반드시 "주의할 점"과 "바로 써볼 프롬프트"를 넣습니다.
  • 민감정보와 진단 결정은 AI가 아닌 의사가 최종 판단한다는 원칙을 반복합니다.
가장 중요한 편집 기준은 "읽고 바로 써먹을 수 있는가"입니다.
